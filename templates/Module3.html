3.1.9 Interactive — Message Timing

Explore timing concepts:

Flow Control — manages the rate of data transmission.

Defines how much data can be sent and at what speed. Prevents a fast sender from overwhelming a slow receiver.
Protocol example: TCP uses a sliding window to control flow — the receiver tells the sender how much buffer space it has.
Response Timeout — how long a device waits for a reply.

If no reply arrives within the timeout period, the sender may retransmit or declare an error.
Example: TCP waits for an ACK. If none arrives within the timeout, it retransmits the segment.
Access Method — determines when a device can send.

Needed on shared media where multiple devices compete to transmit simultaneously.
CSMA/CD — Carrier Sense Multiple Access / Collision Detection (Ethernet wired)
CSMA/CA — Collision Avoidance (Wi-Fi wireless)
Collisions — when two devices send at the same time.

⚡ Proactive
Avoid collisions before they happen
→ CSMA/CA (Wi-Fi)
🔧 Reactive
Recover after a collision occurs
→ CSMA/CD (Ethernet)

Section 3.2 — Types, functions, and how they interact

3.2.1 Protocol Types — Click to explore
Network Comms
IP, TCP, HTTP
Enable two or more devices to communicate over one or more networks. The core of all internet communication.
Network Security
SSH, TLS, SSL
Secure data — provide authentication, data integrity, and encryption. Protect data in transit from eavesdropping.
Routing
RIP, OSPF, BGP
Enable routers to exchange route information, compare path info, and select the best path to any destination.
Service Discovery
DHCP, DNS
Automatic detection of devices or services. DHCP assigns IP addresses; DNS resolves domain names to IPs.

Protocol Suites

Section 3.3 — Groups of protocols that work together

3.3.1 What is a Protocol Suite?

A group of inter-related protocols that together perform a communication function.

🔗Think of it as a team — each protocol has a specific role, and they all work together to get data from A to B
Upper Layers:
Handle user-facing services
(HTTP, DNS, DHCP, FTP)
Lower Layers:
Move data across the network
(IP, TCP, Ethernet)
3.3.2 Protocol Suite History — Click to learn
TCP/IP Internet Protocol Suite
▼
Most common protocol suite. Maintained by IETF (Internet Engineering Task Force). Open standard — free for anyone to use. Powers the entire internet.
OSI Open Systems Interconnection
▼
Developed by ISO and ITU. A reference model — not a protocol itself, but a framework that helps us understand and describe network communication in 7 layers.
AppleTalk Legacy
▼
Proprietary suite released by Apple Inc. Discontinued — Apple devices now use TCP/IP.
Novell NetWare Legacy
▼
Proprietary suite by Novell Inc. Discontinued — replaced by TCP/IP in enterprise environments.
Why TCP/IP Won
✅Open standard — free, no vendor lock-in
🌐Standards-based — endorsed by the networking industry
🔄Interoperable — any device from any vendor can use it
🏛️Approved by standards organizations

OSI Reference Model

Section 3.5 — The 7-layer model — click each layer to explore

Benefits of a Layered Model
Assists protocol design — each layer has defined input/output
Fosters competition — different vendors interoperate
Prevents changes in one layer from affecting others
Provides a common language for networking
3.5.2 OSI Layers — Click any layer
7
Application
DATA
Process-to-process communications. Protocols: HTTP, FTP, DNS, DHCP, SMTP. This is the layer your apps interact with.
6
Presentation
DATA
Common representation of data transferred between app layer services. Handles encryption, encoding, and compression. Example: TLS/SSL.
5
Session
DATA
Manages sessions — setup, maintenance, and teardown of conversations. Manages Unicast, Multicast, Broadcast addressing at the application level.
4
Transport
SEGMENT
Segments, transfers, and reassembles data. Uses port numbers. Protocols: TCP (reliable) and UDP (fast, no guarantee). PDU = Segment.
3
Network
PACKET
Routes data between networks. Uses IP addresses. Protocol: IP (IPv4/IPv6). PDU = Packet. Routers operate at this layer.
2
Data Link
FRAME
Exchanges data frames over common media. Uses MAC addresses. Protocols: Ethernet, Wi-Fi. PDU = Frame. Switches operate here.
1
Physical
BITS
Physical connections — cables, signals, NIC cards. Defines type of media (copper, fiber, wireless). PDU = Bits. This is Module 4!

TCP/IP Model

Section 3.5 — The 4-layer model used by the internet

TCP/IP Layers — Click each layer
4
Application
HTTP DNS DHCP FTP SMTP
Represents data to the user, encoding, and dialog control. Maps to OSI layers 5, 6, and 7. This is where your apps (browser, email, etc.) communicate with the network.
3
Transport
TCP UDP
Supports communication between devices across diverse networks. TCP = reliable, ordered delivery. UDP = fast, no guarantee. Maps to OSI layer 4.
2
Internet
IPv4 IPv6 ICMP
Determines the best path through the network using IP addresses. Routes packets between different networks (LANs, WANs). Maps to OSI layer 3.
1
Network Access
Ethernet Wi-Fi ARP
Controls hardware devices and media. Uses MAC addresses. Covers both the physical connection (cables, signals) and the data link (frames). Maps to OSI layers 1 and 2.
OSI vs TCP/IP Comparison
OSI (7 layers)
7 — Application
6 — Presentation
5 — Session
4 — Transport
3 — Network
2 — Data Link
1 — Physical
TCP/IP (4 layers)
4 — Application
(covers OSI 5,6,7)
3 — Transport
2 — Internet
1 — Network Access
(covers OSI 1,2)
⚠️Key difference: TCP/IP combines OSI layers 1&2 into "Network Access" and OSI layers 5,6,7 into "Application"

Data Encapsulation

Section 3.6 — How data is packaged as it travels down the layers

The Big Idea

Each layer adds its own header (and sometimes trailer) to the data from the layer above — like putting a letter into envelopes.

📮Think of it as a letter: the message → put in an envelope (TCP) → put in a box with address (IP) → put on a truck with a route (Ethernet)
