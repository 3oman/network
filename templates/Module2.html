Cisco IOS Access

Section 2.1 — How to connect and access a Cisco device

Operating System Layers
🖥️ Shell
User interface — CLI or GUI — where you type commands
↕
⚙️ Kernel
Communicates between hardware and software
↕
🔌 Hardware
Physical components — CPU, RAM, ports
GUI vs CLI
GUI — Graphical User Interface
Icons, menus, windows
Examples: Windows, macOS, Android
⚠️ Can crash or fail → not reliable for network devices
CLI — Command Line Interface
Text commands via keyboard
✅ Used for ALL Cisco network devices
More reliable and powerful
2.1.4 Access Methods — How to Connect
🔌
Console
Physical port — direct cable connection. Used for initial setup. Works even with no network config.
✅Best for first-time setup
🔐
SSH
Secure remote CLI over network. Encrypted — passwords and commands are protected.
⭐Recommended for remote access
⚠️
Telnet
Remote CLI over network — but INSECURE. All data (including passwords) sent as plaintext.
❌Avoid in production networks
🖥️Terminal emulation programs: PuTTY, Tera Term, SecureCRT — used to connect via Console or SSH/Telnet

IOS Navigation

Section 2.2 — Moving between command modes

Interactive — Click a mode to learn more

User EXEC Mode

Switch>

Limited — view only. Like "read-only" access.

Only basic monitoring commands allowed. You CANNOT make any changes here.

To enter: default when you first connect
Common commands: show version, ping
To go up: type enable
↕ enable / disable

Privileged EXEC Mode

Switch#

Full access — like "admin" mode. Can view and save config.

Access to ALL commands and features. The # symbol tells you you're here.

To enter: enable from User EXEC
To go down: disable
To go up to config: configure terminal
↕ configure terminal / end

Global Configuration Mode

Switch(config)#

Make changes that affect the entire device.

All device-wide settings are configured here (hostname, passwords, etc.)

To enter: configure terminal from Priv EXEC
To exit to Priv EXEC: exit or Ctrl+Z
Sub-modes from here: Line config, Interface config
↕ line / interface commands

Line Config Mode

Switch(config-line)#

Configure console, SSH, Telnet access.

Enter: line console 0 or line vty 0 15
Exit to global: exit
Exit to priv: end

Interface Config Mode

Switch(config-if)#

Configure switch ports or router interfaces.

Enter: interface vlan 1
Exit to global: exit
Exit to priv: end or Ctrl+Z
💡Quick tip: end or Ctrl+Z always jumps you all the way back to Privileged EXEC from any sub-mode

Command Structure

Section 2.3 — Syntax, help features, and keyboard shortcuts

Command Syntax
ping 192.168.1.1
└─ command └─ argument (user-defined)

ip address 192.168.10.5 255.255.255.0
└─ keyword └─ argument └─ argument
Keyword — predefined word the OS recognizes
Example: ip, address, ping
Argument — value YOU provide
Example: 192.168.1.1, cisco123
❓Type ? at any time to see available commands — this is context-sensitive help
Syntax Conventions
SymbolMeaningExample
boldfaceType exactly as shownenable, ping
italicsReplace with your own valueip-address → you type the actual IP
[x]Optional — can skip it[description text] — optional
{x}Required — must provide it{ip-address} — must enter an IP
x | yChoose one optiontcp | udp — pick one
Keyboard Shortcuts
Tab

Auto-complete a partial command name

↑ Up Arrow

Recall previous commands from history

Ctrl + C

Exit configuration mode → back to Priv EXEC

Ctrl + Z

Same as Ctrl+C — exit any config mode

Ctrl + Shift + 6

Break — abort ping, traceroute, DNS lookups

Space Bar

Show next screen when "--More--" appears

⚡Commands can be shortened! configure terminal → conf t — as long as it's unique

Password Guidelines
More than 8 characters in length
Mix of UPPER, lower, numbers, symbols
Different password for each device
❌ Don't use common words (cisco, admin, password)
❌ Lab passwords like cisco or class are WEAK — never use in production
Device Name Rules
  • Start with a letter
  • No spaces allowed
  • End with a letter or digit
  • Only letters, digits, dashes
  • Less than 64 characters
✅ SW-Floor1
✅ Core-Router-01
❌ My Switch (has space)
❌ -Router (starts with dash)
Banner MOTD

Warning message shown to anyone who tries to log in.

Switch(config)# banner motd #
Enter TEXT message. End with the character '#'.
Authorized Access Only!
Violators will be prosecuted.
#
⚠️The # is the delimiter — marks start and end of message

Save Configurations

Section 2.5 — running-config vs startup-config

The Two Config Files
📋 RAM (Volatile)

running-config

Current active config. Changes take effect immediately. LOST when device powers off!

copy → save
🔒 NVRAM (Non-Volatile)

startup-config

Saved config. Loaded when device boots. Survives power off and reloads.

-- Save running config to startup --
Switch# copy running-config startup-config
Destination filename [startup-config]? [Enter]
Building configuration...
[OK]
Recover / Undo Changes
SituationSolution
reloadReload device — running-config cleared, startup-config reloads. Causes brief downtime!
erase startup-configWipes the saved startup-config from NVRAM. Then reload to reset everything.
Remove commands individuallyAdd no before any command to undo it. Example: no hostname
⚠️If bad config was NOT saved yet → just reload and it's gone. If it WAS saved → must erase startup-config first!
Full Save Workflow
-- Check current config --
Switch# show running-config

-- Save it --
Switch# copy running-config startup-config

-- Verify it saved --
Switch# show startup-config

-- If you need to erase and restart --
Switch# erase startup-config
Switch# reload

Each number = 0 to 255 — called dotted decimal notation

IP Address
192.168.10.5
Unique ID of your device on the network
Subnet Mask
255.255.255.0
Tells which part is network vs host
Default Gateway
192.168.10.1
Router address — used to reach other networks
IPv6 Address
2001:0db8:85a3:0000:0000:8a2e:0370:7334
  • 128 bits long (vs IPv4's 32 bits)
  • Written in hexadecimal, groups of 4 digits separated by :
  • Not case-sensitive — upper or lowercase OK
  • The future replacement for IPv4
🔄IPv6 uses DHCPv6 and SLAAC for automatic address assignment (instead of DHCP)
Switch Virtual Interface (SVI) Configuration

To manage a switch remotely, you need to assign it an IP address on the virtual interface (VLAN 1):

Switch# configure terminal
Switch(config)# interface vlan 1
Switch(config-if)# ip address 192.168.1.10 255.255.255.0
Switch(config-if)# no shutdown
%LINEPROTO-5-UPDOWN: Line protocol on Interface Vlan1, changed state to up
Switch(config-if)# exit
Switch(config)# ip default-gateway 192.168.1.1
✅no shutdown enables the interface — without it the interface stays down!
Manual IP (Windows PC)
Control Panel
→ Network Sharing Center
→ Change adapter settings
→ Right-click → Properties
→ IPv4 Properties
→ Enter IP, Mask, Gateway manually
Automatic IP (DHCP)
Same path as manual but:
→ Select "Obtain an IP address automatically"
→ Select "Obtain DNS server address automatically"

✅ Default setting on most devices
